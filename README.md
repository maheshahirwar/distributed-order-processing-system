# Distributed Order Processing System

A scalable microservices-based order processing platform built using event-driven architecture and distributed system design principles.

## Overview

This project simulates a real-world e-commerce order processing workflow involving inventory management, payment processing, shipment handling, and customer notifications.

The platform is designed for scalability, fault tolerance, and high availability.

## Features

* Order Management
* Inventory Service
* Payment Service
* Shipment Service
* Notification Service
* Event-Driven Architecture
* Kafka Messaging
* Redis Caching
* Distributed Transactions
* Fault Tolerance
* Retry Mechanisms
* Dead Letter Queues

## Tech Stack

* Java / Spring Boot
* Kafka
* Redis
* MySQL
* Docker
* Kubernetes
* AWS

## Architecture

Order Service → Kafka → Inventory Service → Payment Service → Shipment Service → Notification Service
